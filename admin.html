SourceKey / contextEN (base)ES (req)ITDEFRFill

This is where you change any user-facing text anywhere in the app — a tip-frame line, the bundle frame, a nav label, a wheel descriptor, a cheat sheet, a map description — most often to fix the accuracy of what's taught. Use the Source filter to jump to a zone. EN is the base; ES required for launch; IT/DE/FR optional (fall back to EN). When you change the base text, existing translations are flagged stale until re-checked, so a corrected fact can't go out with an out-of-date translation. Every edit is logged with a reason.

Run all three gates before publishing. A change can only be published when the structural gate passes, the required-language gate passes, and an assistant has signed off.

Canonical multilingual schema

Every user-facing string is a keyed entry. EN is the base; ES required; IT/DE/FR optional with English fallback. This is the shape every wheel/cheat/UI dataset should adopt before the big build so adding a language is just filling slots — never re-architecting.

{
  "key": "bubbles.brut-nv.name",
  "context": "Sparkling wheel · descriptor name",
  "en": "Brut Non-Vintage",   // base (required)
  "es": "Brut sin añada",     // required for launch
  "it": "", "de": "", "fr": ""// optional → fall back to EN
}

Runtime helper (drop into wheels):

function t(v, lang){ if(v==null) return '';
  if(typeof v==='string') return v;          // legacy plain string
  return v[lang] || v.en || Object.values(v).find(Boolean) || ''; }
